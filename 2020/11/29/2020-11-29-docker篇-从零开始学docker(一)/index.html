编写dockerFile 构建docker镜像

构建命令:  docker image build
Dockerfile 通过环境变量修改
参数 使用$param 或者 ${param}
${param+word} 变量不为空使用word
${param-word} 变量为空使用默认值word

Dockerfile语法

  1. 注释信息 以# 开头
  2. 命令
  3. 语句顺序执行
  4. 对大小写不敏感 约定俗称 关键字使用大写
  5. 第一个非注释行必须为FROM 命令 通过一个已有镜像制作

*创建专门Dockerfile文件 D必须大写
*所有需要文件必须在这个在这个文件下的子目录中

  • 创建 .dockeringore 文件 可以在打包镜像忽略部分指定文件

Dockerfile指令
\ 反斜杠用于换行

  1. FROM 【使用：FROM busybox：latest 】镜像来源

  2. MAINTANIER 【使用：MAINTANIER “zztzzt@email.com“】镜像作者

  3. LABEL【使用: LABEL = = … 】

  4. COPY 【使用：COPY 原文件路径 原文件路径 目标文件路径 或COPY “原文件路径””原文件路径””目标文件路径”】用于从docker主机复制文件到创建的新镜像 复制目录时 将目录内容复制到指定位置 通常使用第二种方式 原路径不会被复制 只复制内容。且原目录必须以 / 结尾

  5. 由于docker采用分层结构所以 每条语句都会产生新的层 ，所以尽量将语句放在一起写

  6. ADD 使用方法同COPY 但是 如果原文件是tar压缩文件（或下载链接文件） 会被自动展开为到路径下（url地址不会展开压缩文件）

  7. WORKDIR 指定目录 之后的语句可以使用 ./ 指定为当前目录 直到下一个WORKDIR定义出现

  8. VOLUME 仅可指定挂载点 【VOLUME /data/***/】

  9. EXPOSE 用于为容器打开指定要监听的端口实现与外部通信【EXPOSE 8080/tcp 8080/udp】 tcp 和udp二选一 默认为tcp 该端口为可以暴露的接口 启动时并不会直接暴露

  10. ENV 用于为镜像定义所需环境变量 可被dockerfile文件之后的指令所使用 【使用：ENV = = ….. 】\反斜杠在此处还可用于转义空格 和docker run –env 不同 此处是用在build过程中设置好的环境变量

  11. RUN 运行sh命令在build过程【使用 ：RUN 命令1 && 命令2 】

  12. CMD 运行sh命令在启动过程 【使用 ：CMD 命令1 && 命令2】
    Cmd 命令可以被docker run 启动时的命令替换掉
    定义多个cmd仅仅最后一个生效
    数组写法不可使用环境变量参数

  13. ENTRYPOINT
    和CMD相比 默认情况命令不会类替换掉 如要替换 使用
    Docker container run –entrypoints ““ container
    定义多个ENTRYPOINT 最后一个生效
    和CMD同时存在情况下 CMD会被当做参数交给ENTRYPOINT 执行
    可通过添加entrypoint.sh 来让ENTRYPOINT 执行该文件 带入参数

  14. USER 在某个用户下运行

  15. HEALTHCHECK 健康检查

  16. SHELL 指定程序默认要使用的shell程序
    Eg:
    SHELL[“shell工具”,”参数”]

  17. STOPSIGNAL 停止进程

  18. ARG 语句向dockerfile 中传递参数(build 过程使用)
    Eg： ARG [=]
    Docker file中
    ARG latest_tag=”latest”
    EXPOSE ${latest_tag}
    执行
    Docker image build –arg latest_tag=”7” -t centos:v0.1.1 ./

  19. ONBUILD 用于dockerfile中定义触发器
    在其他人使用此镜像制作新镜像时执行dockerfile的指令
    Eg : ONBUILD ADD “tomcat.zip” “/data”（dockerfile的指令）
    自己制作镜像时不执行该语句 在其他基于此镜像制作新镜像时 会执行

Docker Compose

docker资源限制（默认无限制）

内存为非可压缩资源 限制避免oom内存溢出
发生oom会触发进程被按照相对内存的使用量依次强制杀掉（kill -9 pid） 直到没有异常
–oom-score-adj int Tune host’s OOM preferences (-1000 to 1000)
调整优先级 尽量避免被杀掉
Ram 物理内存
Swap 交换内存
-m 设置物理内存使用量
–memory-swap * 设置交换内存使用量 必须先限制物理内存 -m

–oom-kill-disable 禁止发生oom时被kill 必须先限制物理内存 -m

--memory-swap --memory 功能
正数S 正数M 容器可用空间为S其中 RAM为M swap为（S-M）若S=M 则无可用swap资源
0 正数M 相当于未设置swap
unset 正数M 若docker host 开启swap 容器可用则为2M
-1 正数M 若docker host 开启swap 则容器可使用主机全部swap资源

*NOTE:在容器中使用free命令查看的swap并不具有展示空间的指示意义。

Cpu 为可压缩性资源
进程优先级交由cfs 内核调度器调整
进程分2类：
Cpu密集型
IO密集型
–cpu-period int Limit CPU CFS (Completely Fair Scheduler) period
–cpu-quota int Limit CPU CFS (Completely Fair Scheduler) quota
–cpu-rt-period int Limit CPU real-time period in microseconds限制cpu使用时间
–cpu-rt-runtime int Limit CPU real-time runtime in microseconds
-c, –cpu-shares int CPU shares (relative weight)
–cpus decimal Number of CPUs
–cpuset-cpus string CPUs in which to allow execution (0-3, 0,1)运行在那个cpu上 0开始
–cpuset-mems string MEMs in which to allow execution (0-3, 0,1)
